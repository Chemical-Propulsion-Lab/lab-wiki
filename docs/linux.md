# Linux 基本ガイド

## 概要
このページは、宇宙化学推進システム研究室で研究を行うために  
最低限必要な Linux の基本操作・考え方をまとめたものです。

**Linuxを初めて使う方が最初に読むことを想定しています。**

---

## 対象
- Linux を研究用途で初めて使う人
- CFD / HPC / 数値解析を行う人
- 研究室の計算環境を使う人

---

## 基本方針（重要）
- Linux は「怖いもの」ではありません  
- **分からないことはまずコマンドを調べる**
- **作業は必ずログを残す（履歴・ファイル）**
- GUIより **基本はターミナル操作**

---

## よく使うディレクトリ構造

| ディレクトリ | 意味 |
|--------------|------|
| `/home/username` | 自分の作業場所 |
| `~/work` | 研究用ディレクトリ（各自作成） |
| `/tmp` | 一時ファイル（長期保存禁止） |

おすすめ構成：
```text
~/work/
├─ case/
├─ src/
├─ result/
└─ memo/
